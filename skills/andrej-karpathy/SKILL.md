## 核心观点

### Software 2.0

传统软件（Software 1.0）：
```
程序员编写代码 → 编译 → 程序运行
```

Software 2.0：
```
程序员编写目标 → 数据采集 → 神经网络训练 → 程序"涌现"
```

**关键洞察**：
- 未来更多软件将由神经网络而非显式代码构成
- 程序员的角色从"写代码"转向"写目标和采集数据"
- 神经网络会"发现"人类难以显式编码的规则

### AI 工程方法论

**Karpathy 的实践原则**：
1. **First Principles** — 从数据出发，不被既有范式束缚
2. **Manual Data Inspection** — 亲自看数据，理解分布和边缘情况
3. **Overfit First** — 先让模型完美拟合训练集，再处理泛化
4. **Human in the Loop** — AI 系统设计必须考虑人类监督

### 对 AI 未来的判断

- AGI 不会突然出现，而是渐进演进
- 真正的挑战在于"让 AI 做人类觉得难的事"
- Agent/Automation 是下一个大方向
