## 核心操作

### 客户接入 (Client Onboarding)

**流程**:

简介到达（音频、邮件、文档） → 提取范围、预算、时间线 → 生成结构化简介 → 标记危险信号（范围蔓延、不切实际的截止日期） → 创建客户文件夹

**输入**:

- 客户邮件
- 项目简介文档
- 音频说明
- 直接消息

**输出**:

- 结构化简介文件：`~/agency/clients/[name].md`
- 客户索引更新：`~/agency/clients/index.md`
- 项目文件夹：`~/agency/projects/[project-name]/`

**危险信号检查清单**:

- [ ] 范围蔓延（要求超出原始简介）
- [ ] 不切实际的截止日期
- [ ] 预算不足
- [ ] 沟通风格不匹配
- [ ]  Roles and responsibilities unclear

### 定价和提案 (Pricing and Proposals)

**流程**:

给定范围 → 应用配置中的价目表 → 计算带有复杂性乘数的估算 → 生成提案PDF → 与历史类似项目比较

**定价配置**: `~/agency/config.md`

```markdown
# 费率卡

## 角色费率

| 角色 | 小时费率 | 日费率 | 项目费率 |
|------|----------|---------|----------|
| 项目经理 | $100 | $700 | 按项目 |
| 开发者 | $80 | $560 | 按项目 |
| 设计师 | $70 | $490 | 按项目 |
| 测试者 | $60 | $420 | 按项目 |

## 复杂性乘数

| 复杂性 | 乘数 | 描述 |
|--------|------|------|
| 简单 | 1.0x | 标准实施 |
| 中等 | 1.5x | 一些技术挑战 |
| 复杂 | 2.0x | 重大技术挑战 |
| 非常复杂 | 3.0x | 前沿技术 |

## 利润率

目标利润率：30%
最低利润率：15%
```

**提案模板**: `~/agency/templates/proposal-template.md`

**输出**:

- 提案PDF：`~/agency/projects/[project-name]/proposal.pdf`
- 定价明细：`~/agency/projects/[project-name]/pricing-breakdown.md`

### 项目跟踪 (Project Tracking)

**流程**:

维护所有活跃项目的统一看板 → 截止日期警报 → 检测停滞项目 → 按客户生成每周状态

**项目状态**:

| 状态 | 描述 |
|------|------|
| 咨询中 | 初始讨论，尚未签约 |
| 进行中 | 活跃工作 |
| 审查中 | 等待客户反馈 |
| 修订中 | 根据反馈修订 |
| 已完成 | 项目完成并交付 |
| 已存档 | 项目存档 |

**项目跟踪文件**: `~/agency/projects/INDEX.md`

```markdown
# 项目索引

| 项目名称 | 客户 | 状态 | 截止日期 | 进度 |
|---------|------|--------|----------|------|
| 网站重建 | Acme Corp | 进行中 | 2026-06-01 | 60% |
| 移动应用 | Beta Inc | 审查中 | 2026-05-15 | 80% |

总项目数：2个活跃，5个已完成
```

**警报规则**:

- 截止日期前7天：警告
- 截止日期前3天：紧急
- 截止日期已过：逾期
- 无活动14天：停滞

### 交付物工作流 (Deliverables Workflow)

**流程**:

转换粗略笔记/输入 → 结构化交付物 → 对照简介审查 → 适应多种格式（如需要）

**交付物类型**:

| 类型 | 格式 | 示例 |
|------|------|------|
| 报告 | PDF, DOCX | 项目状态报告、审计报告 |
| 代码 | 仓库 | GitHub仓库、源代码 |
| 设计 | Figma, PDF | UI设计、品牌指南 |
| 文档 | MD, DOCX | API文档、用户手册 |
| 数据 | CSV, XLSX | 分析报告、数据集 |

**交付物检查清单**:

- [ ] 对照简介验证
- [ ] 符合质量标准
- [ ] 正确格式化
- [ ] 包含所有必需元素
- [ ] 已审查无错误
- [ ] 已交付给客户

**交付物存储**: `~/agency/projects/[project-name]/deliverables/`

### 团队协调 (Team Coordination)

**流程**:

分配任务 → 跟踪进度 → 促进沟通 → 解决冲突 → 绩效评估

**团队结构配置**: `~/agency/config.md`

```markdown
# 团队结构

## 内部团队
- 项目经理：Alice
- 首席开发者：Bob
- 设计师：Charlie

## 外部合作者
- 测试者：David（合同工）
- 内容创作者：Eve（合同工）

## 联系信息
- Alice：alice@example.com
- Bob：bob@example.com
- Charlie：charlie@example.com
```

**任务分配模板**:

```markdown
# 任务分配

## 任务详情
名称：<任务名称>
分配给：<团队成员>
截止日期：<日期>
优先级：<高/中/低>

## 描述
<任务描述>

## 交付物
<交付物列表>

## 依赖
<依赖列表>
```

### 代理机构类型特定 (Agency Type Specifics)

**支持的类型**:

| 类型 | 特定工作流 |
|------|-------------|
| 营销代理机构 | 内容日历、活动跟踪、ROI分析 |
| 开发代理机构 | 代码审查、测试自动化、部署管道 |
| 设计代理机构 | 设计系统、资产库、版本控制 |
| 咨询代理机构 | 研讨会、评估、建议 |
| 内容代理机构 | 编辑日历、内容库、分发跟踪 |

**类型特定配置**: `~/agency/config.md`

```markdown
# 代理机构类型：开发代理机构

## 特定工作流
- 代码审查：每个PR都需要审查
- 测试自动化：CI/CD管道
- 部署管道：自动部署到暂存和生产

## 特定工具
- GitHub：代码仓库
- Jira：项目管理
- Jenkins：CI/CD
```

### 学习系统 (Learning System)

**流程**:

收集反馈 → 识别模式 → 更新模板和知识库 → 改进流程

**反馈来源**:

- 客户反馈
- 团队反馈
- 项目事后分析
- 自我反思

**知识库结构**: `~/agency/knowledge/`

```
~/agency/knowledge/
├── lessons-learned/     # 项目事后分析
├── best-practices/      # 最佳实践
├── templates/           # 改进模板
└── case-studies/        # 案例研究
```

**学习循环**:

1. **收集**: 从每个项目收集反馈
2. **分析**: 识别模式和趋势
3. **更新**: 更新模板、流程、知识库
4. **分享**: 与团队分享学习
5. **改进**: 将学习应用到未来项目

## 工作空间结构

```
~/agency/
├── clients/           # 每个客户一个文件
│   ├── index.md       # 客户列表（含状态）
│   └── [name].md      # 客户档案、历史、偏好
├── projects/          # 活跃项目跟踪
│   ├── INDEX.md       # 项目索引
│   └── [project]/    # 单个项目文件夹
├── templates/         # 可重用的提案、简介、报告
├── knowledge/         # SOP、学习、案例研究
└── config.md          # 费率、利润、团队结构
```

## 关键规则

- **绝不在没有人类批准的情况下** 发送提案或与客户沟通
- **跟踪时间/成本 vs 估算** - 当项目亏损时警报
- **从纠正中学习** - 更新模板和知识库
- **跨会话维护客户上下文** - 参考历史
