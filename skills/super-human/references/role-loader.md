## 角色加载流程

### 1. 识别需求

**输入**: 用户请求 + 上下文

**处理**:
```
分析用户请求 → 提取关键词 → 匹配角色类型
```

**示例**:
```
用户："作为安全工程师，请审查这段代码"
关键词：安全工程师
匹配角色：engineering-security-engineer
```

### 2. 查找角色文件

**标准路径**: `/root/.openclaw/agency-agents-zh/{domain}/{role-name}.md`

**查找策略**:
1. 精确匹配：角色名称完全一致
2. 模糊匹配：关键词匹配
3. 推荐匹配：如果找不到，推荐最相关的角色

**示例**:
```bash
# 精确匹配
cat /root/.openclaw/agency-agents-zh/engineering/engineering-security-engineer.md

# 模糊匹配
grep -r "安全" /root/.openclaw/agency-agents-zh/

# 查看所有角色
cat /root/.openclaw/agency-agents-zh/AGENT-LIST.md
```

### 3. 加载角色定义

**角色文件结构**:
```markdown
---
name: 角色名称
domain: 领域
expertise: 专业领域列表
workflow: 工作流程
deliverables: 交付物标准
---

# 角色名称

## 专业概述

## 核心能力

## 工作流程

## 交付物标准

## 与中国市场相关的特定知识
```

**加载到上下文**:
```
1. 读取角色文件
2. 提取核心能力、工作流程、交付物标准
3. 将角色行为模式注入当前会话上下文
4. 按角色标准执行任务
```

### 4. 执行任务

**执行规则**:
- 严格按照角色的工作流程执行
- 交付物符合角色的专业标准
- 使用角色的专业术语和思维方式
- 如有需要，可以参考该领域的中国市场营销

**示例**:
```
[已加载：安全工程师角色]

用户：请审查这段代码的安全漏洞
AI：[按安全工程师的工作流程]
   1. 静态分析：检查常见漏洞（OWASP Top 10）
   2. 数据流追踪：检查输入验证和输出编码
   3. 权限审查：检查认证和授权逻辑
   4. 生成安全审计报告（符合专业标准）
```

## 角色组合使用

### 多角色协作

**场景**: 复杂任务需要多个专家协作

**工作流程**:
```
1. 识别所需的所有角色
2. 按顺序加载每个角色
3. 定义角色间的协作接口
4. 执行多角色协作任务
```

**示例**: 开发一个安全的电商网站
```
所需角色：
1. 安全工程师 - 定义安全需求
2. 后端架构师 - 设计系统架构
3. 前端开发者 - 实现用户界面
4. UI设计师 - 设计用户体验

协作流程：
安全工程师 → 后端架构师 → 前端开发者 → UI设计师
```

### 角色冲突解决

**冲突类型**:
1. 工作流程冲突：角色A要求先设计，角色B要求先开发
2. 交付物标准冲突：角色A的交付物标准与角色B不兼容
3. 优先级冲突：角色A认为安全最重要，角色B认为性能最重要

**解决策略**:
1. **优先级排序**：用户明确指定优先级
2. **分阶段执行**：不同阶段使用不同角色
3. **专家决策**：如果冲突无法解决，咨询用户

## 动态角色创建

### 如果角色库中不存在所需角色

**创建流程**:
```
1. 定义角色名称和专业领域
2. 从现有角色中借鉴相似能力
3. 创建新的角色定义文件
4. 保存到用户自定义角色目录：~/memory/custom-roles/
5. 记录创建原因和用途
```

**示例**: 创建"微信小程序优化专家"
```
借鉴角色：
- 前端开发者（性能优化）
- UI设计师（小程序设计规范）
- 市场营销（微信生态）

创建文件：~/memory/custom-roles/wechat-miniprogram-optimizer.md
```

## 角色性能优化

### 缓存机制

**缓存内容**:
- 最近使用的角色定义
- 角色的工作流程模板
- 角色的交付物标准

**缓存策略**:
- LRU缓存：最近最少使用淘汰
- 按领域缓存：常用领域的角色优先缓存
- 用户偏好缓存：用户常用的角色永久缓存

### 预加载机制

**触发条件**:
- 用户历史中频繁使用某个角色
- 当前任务明显需要某个角色

**预加载策略**:
```
识别到需要角色 → 后台预加载角色定义 → 用户请求时立即使用
```

### 专属角色示例

**小红书运营专家**:
- 种草笔记策略
- 达人合作
- 平台规则合规

**抖音内容策略师**:

**微信小程序开发者**:
- 小程序开发规范
- 微信生态集成
- 用户体验优化

**飞书集成开发者**:
- 飞书应用开发
- 机器人集成
- 企业工作流自动化

**钉钉集成开发者**:
- 钉钉工作台开发
- 智能助手集成
- 企业级应用开发

### 使用建议

**优先使用中国市场角色**:
- 当用户业务涉及中国市场时
- 当用户提到中国本土平台（微信、抖音、小红书等）
- 当用户需要符合中国法规和习俗的建议

**与国际角色结合**:
- 技术类角色可以使用国际最佳实践
- 商业类角色结合中国市场营销
- 设计类角色兼顾国际趋势和中国审美

## 调试和日志

### 角色加载日志

在 `~/memory/super-human/role-logs/` 中记录：
- 每次角色加载的决策过程
- 角色文件位置
- 加载时间和性能
- 角色执行效果和用户反馈

### 角色性能评估

**评估指标**:
- 角色加载时间
- 角色执行准确度
- 用户满意度
- 交付物质量

**优化策略**:
- 对高性能角色，优化加载速度
- 对低性能角色，更新角色定义或移除
- 定期审查角色库，移除过时角色
