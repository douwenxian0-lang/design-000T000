# 记忆系统协同（来自 memory）

本文档定义内置记忆与扩展记忆系统的协同工作机制。

---

## 双轨记忆架构

```
内置Agent记忆          扩展记忆系统 (~/memory/)
┌─────────────────┐        ┌─────────────────────────┐
│ MEMORY.md           │        │ 无限分类存储                │
│ memory/ (daily logs)│   +    │ 任何你想要的结构            │
│ 基本回忆能力        │        │ 完美组织                    │
└─────────────────┘        └─────────────────────────┘
         ↓                                  ↓
   快速上下文                       深度和长期存储
   (自动工作)                      (按需扩展)
```

**核心原则**: 本系统是**平行互补**，不是替代。两个系统一起工作。

---

## 内置记忆 vs 扩展记忆

| 存储在内置记忆（快速上下文） | 存储在扩展记忆（深度存储） |
|--------------------------------|----------------------------------|
| 当前项目状态 | 详细项目历史 |
| 关键联系人快速参考 | 完整联系人档案 |
| 最近决策 | 所有决策推理 |
| 快速事实 | 领域知识库 |
| 摘要 | 集合、清单 |

**规则**: 内置记忆用于快速上下文。扩展记忆用于深度和可扩展性。

## 设置流程

### 首次使用设置

1. **选择类别** - 询问用户想存储什么
2. **配置文件结构** - 为记忆系统定义类别
3. **加载角色库** - 确认agency-agents-zh已安装
4. **设置设计偏好** - 让design技能自动学习，或手动配置

### 用户定义的类别

**在设置期间，询问他们想存储什么。根据他们的需求创建类别：**

| 他们说... | 创建 |
|-------------|--------|
| "我有很多项目" | `~/memory/projects/` |
| "我遇到很多人" | `~/memory/people/` |
| "我想跟踪决策" | `~/memory/decisions/` |
| "我在学习[主题]" | `~/memory/knowledge/[topic]/` |
| "我收集[东西]" | `~/memory/collections/[things]/` |

**无预设结构。** 构建他们需要的东西。

## 写入策略

### 立即写入

**当用户输入重要信息时：**
1. 写入 ~/memory/ 中的适当文件
2. 更新类别 INDEX.md
3. 然后回应

**不要等待。不要批处理。立即写入。**

### 分类存储

**项目信息** → `~/memory/projects/[project-name].md`
**联系人信息** → `~/memory/people/[person-name].md`
**决策推理** → `~/memory/decisions/YYYY-MM-DD-[decision].md`
**领域知识** → `~/memory/knowledge/[topic]/[subtopic].md`
**集合** → `~/memory/collections/[collection-name]/[item].md`

## 搜索策略

### 对于小记忆（<50文件)

```bash
# Grep足够快
grep -r "keyword" ~/memory/
```

### 对于大记忆（50+文件）

通过索引导航：
```
1. ~/memory/INDEX.md → 找到类别
2. ~/memory/{category}/INDEX.md → 找到项目
3. ~/memory/{category}/{item}.md → 读取详情
```

### 对于巨大记忆（500+文件）

使用语义搜索（如果可用），或分层索引：
```
~/memory/projects/INDEX.md → "web projects in web/"
~/memory/projects/web/INDEX.md → "alpha project"
~/memory/projects/web/alpha.md → details
```

## 同步机制（可选）

### 从内置记忆同步

**如果用户想从内置记忆复制某些信息：**

```
~/memory/sync/
├── preferences.md    # 从内置同步
└── decisions.md      # 从内置同步
```

**同步是单向的：** 内置 → 本系统。**永远不要修改内置记忆。**

### 同步配置

在 `~/memory/config.md` 中配置同步设置：

```markdown
# 同步配置

## 从内置记忆同步
- [x] 用户偏好 → ~/memory/sync/preferences.md
- [x] 决策历史 → ~/memory/sync/decisions.md
- [ ] 项目笔记 → ~/memory/sync/project-notes.md

## 同步频率
每日自动同步（通过cron）

## 冲突解决
内置记忆优先（内置是真实来源）
```

## 索引维护

### 每个类别有一个索引

**每个文件夹获得一个 INDEX.md，列出内容：**

```markdown
# 项目索引

| 名称 | 状态 | 更新 | 文件 |
|------|--------|---------|------|
| Alpha | 活跃 | 2026-02 | alpha.md |
| Beta | 暂停 | 2026-01 | beta.md |

总计：2个活跃，5个已存档
```

**索引保持小巧（<100条目）。** 当满时，分裂成子类别。

### 更新索引

**当添加、修改或删除条目时，立即更新INDEX.md：**

```bash
# 添加新项目后
echo "| NewProject | 活跃 | $(date +%Y-%m-%d) | newproject.md |" >> ~/memory/projects/INDEX.md
```

## 扩展策略

### 通过分裂扩展

**当类别增长很大时：**
- INDEX.md > 100条目 → 分裂成子类别
- 为每个子类别创建子INDEX.md
- 根INDEX.md指向子类别

```
~/memory/projects/
├── INDEX.md           # "参见 active/, archived/"
├── active/
│   ├── INDEX.md       # 30个活跃项目
│   └── ...
└── archived/
    ├── INDEX.md       # 200个已存档项目
    └── ...
```

### 存档策略

**当项目完成或联系人不再活跃时：**
1. 移动到 `archived/` 子目录
2. 更新 INDEX.md
3. 保留历史记录以供参考

## 维护计划

### 每周（5分钟）

- 如果添加了条目，更新 INDEX.md 文件
- 存档完成/不活跃的项目

### 每月（15分钟）

- 审查类别大小
- 将大类别分裂
- 移除过时条目

### 当记忆变慢时

- 检查 INDEX.md 大小（保持<100行）
- 将大类别分裂成子类别
- 存档旧内容

## 常见问题

### 问题：修改了内置记忆

**症状**: 意外修改了AGENT的 MEMORY.md 或 workspace memory/

**修复**:
1. 从备份恢复（如果存在）
2. 将来，永远不要触摸内置记忆
3. 本系统在 ~/memory/ 中，平行工作

### 问题：没有索引

**症状**: 找不到存储的信息

**修复**:
1. 立即为所有类别创建 INDEX.md
2. 没有索引，查找事情需要搜索所有文件
3. 总是维护索引

### 问题：一个巨大类别

**症状**: 一个文件夹中有500个条目，搜索变慢

**修复**:
1. 将大类别分裂成子类别
2. 创建子INDEX.md
3. 将根INDEX.md指向子类别

### 问题：同步了一切

**症状**: ~/memory/sync/ 中有太多文件，许多不相关

**修复**:
1. 只同步需要此处组织的内容
2. 不要复制所有内置记忆
3. 有选择性地同步

### 问题：等待写入

**症状**: 信息丢失，因为Agent批处理写入

**修复**:
1. 当用户输入信息时，立即写入
2. 不要等待或批处理
3. 先写入，然后回应

## 安全与隐私

### 数据位置

- 所有数据在用户机器上的 `~/memory/` 中
- 不需要外部服务
- 没有网络请求

### 本技能不：

- 访问内置Agent记忆（仅读取是否同步）
- 发送数据到任何地方
- 存储凭据（永远不要在记忆中存储秘密）
